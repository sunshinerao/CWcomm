# CWcomm

CWcomm 是一个面向**活动现场实时收音-转译-播放**场景的跨端网页应用（电脑 / 手机 / iPad）。

> 当前阶段：**需求梳理与方案规划阶段（Pre-implementation）**。本仓库暂不提交业务代码，先明确功能范围、技术路线、交付计划与风险控制。

---

## 1. 产品愿景

在会议、展会、演讲、培训等场景中，现场音频由采集设备接入系统，系统通过 AI 实时识别与翻译，观众在各类终端自由选择语种收听，从而降低跨语言沟通成本。

---

## 2. 核心目标

- **低延迟**：现场发言到终端播放尽量控制在可接受范围（目标 2~6 秒，按网络与模型性能动态波动）。
- **多语言**：支持至少 3 种目标语种并可扩展。
- **跨端适配**：PC 浏览器、手机浏览器、iPad 浏览器获得一致核心体验。
- **高可用**：网络波动时具备降级能力（如切换文本字幕、重连、缓冲策略）。
- **可运营**：支持活动级管理（活动创建、语种配置、接入码/二维码加入）。
- **可集成**：活动信息可本地创建或通过气候周网站 OpenAPI 同步，用户可复用气候周账号登录。

---

## 3. 关键用户角色

1. **主办方/管理员**
   - 创建活动、配置语种、管理设备接入与监控状态。
2. **技术支持/音频工程师**
   - 接入收音设备，确认音频质量与延迟。
3. **终端听众**
   - 扫码进入网页，选择语言，收听翻译结果。

---

## 4. MVP 功能范围（第一阶段）

### 4.1 活动管理
- 创建活动（名称、时间、默认语种、可选语种）。
- 支持从气候周网站 OpenAPI 拉取活动信息并同步到本系统。
- 生成活动加入链接与二维码。
- 活动状态：未开始 / 进行中 / 已结束。

### 4.2 音频接入
- 支持现场采集端接入音频流。
- 基础音频状态监控：输入电平、静音检测、连接状态。

### 4.3 实时识别与翻译
- 音频 -> ASR（自动语音识别） -> 中间文本。
- 文本 -> 多语种翻译（可并行翻译多个目标语种）。
- 输出结构含时间戳与片段 ID（便于同步与容错）。

### 4.4 终端播放与展示
- 终端用户选择目标语种。
- 播放翻译语音（TTS）或优先使用低延迟语音通道。
- 同步展示字幕（原文 + 译文，作为兜底模式）。

### 4.5 账号与集成
- 支持本系统账号体系与气候周网站用户体系集成登录（建议 OAuth2/OIDC）。
- 支持首次登录用户自动建档与角色映射（听众/管理员）。

### 4.6 稳定性能力
- 自动重连。
- 网络弱时降级策略（优先字幕，降低音频码率，延迟提示）。
- 简单端到端监控指标（连接数、平均延迟、失败率）。

---

## 5. 非功能性要求

- **性能**：
  - 首屏加载（4G 条件）< 3s（核心页面）。
  - 终端切换语种生效 < 1s（不含模型冷启动）。
- **兼容性**：
  - Chrome/Safari/Edge 最新稳定版。
  - iOS Safari、Android Chrome。
- **安全合规**：
  - 全链路 HTTPS/WSS。
  - 活动访问鉴权（短期 token / 活动码）。
  - 音频与文本日志脱敏与保留策略可配置。

---

## 6. 技术路线（建议）

> 根据实时性需求，建议采用“**WebRTC + 实时 AI 服务 + WebSocket 控制信令**”混合架构。

- **WebRTC（可选核心）**
  - 用于低延迟音频分发，适合现场高并发收听。
- **WebSocket**
  - 承载控制指令、状态推送、字幕增量更新。
- **AI 链路**
  - ASR（流式） -> 翻译（增量） -> TTS（流式）
- **前端**
  - 响应式 Web App（PWA 可后续加入），统一一套代码适配 PC/手机/iPad。
- **后端**
  - 会话编排、语种路由、鉴权、监控、日志。
- **外部集成层**
  - 对接气候周 OpenAPI（活动数据同步）与统一身份登录（SSO）。

详见：`docs/requirements-and-plan.md`

## 7. 里程碑规划（建议）

- **M0（需求确认）**：场景边界、MVP 清单、指标定义。
- **M1（技术验证）**：完成端到端 PoC（单语种 + 字幕）。
- **M2（MVP）**：多语种选择、音频播放、活动管理最小闭环。
- **M3（试点）**：真实活动压测与问题修复。
- **M4（上线）**：运维监控、告警、手册与发布。
