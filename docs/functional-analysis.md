# CWcomm 功能分析（第二轮细化）

> 目的：在既有需求文档基础上，把功能拆解到“可设计、可评审、可验收”粒度，作为架构设计与 API 草案的输入。

---

## 1. 分析范围与原则

- 仅覆盖 MVP 范围内的核心闭环：活动管理、实时转译分发、终端收听、账号集成。
- 对“气候周 OpenAPI 活动同步”和“气候周账号登录集成”做显式规则化。
- 每个功能模块同时给出：
  - 业务目标
  - 关键规则
  - 失败/降级策略
  - MVP 验收点

---

## 2. 功能域拆解

### 2.1 活动域（Activity Domain）

**业务目标**
- 提供活动从创建到结束的全生命周期管理。
- 支持两种来源：CWcomm 本地创建、气候周 OpenAPI 同步。

**核心对象**
- `Activity`：活动主数据（标题、时间、地点、状态、来源）。
- `ActivityLanguage`：活动支持语种。
- `AccessEntry`：加入信息（加入码、链接、二维码）。

**关键规则**
- 活动状态机：`DRAFT -> READY -> LIVE -> ENDED -> ARCHIVED`。
- 仅 `READY/LIVE` 状态允许终端加入。
- `LIVE` 状态下禁止修改影响实时链路的关键参数（源语种、音频输入配置）。

**降级策略**
- 气候周同步失败时，保留上次成功快照并标记“同步滞后”。

**MVP 验收点**
- 可创建、发布、开始、结束活动。
- 可生成可用加入链接与二维码。

### 2.2 音频与转译域（Realtime Pipeline Domain）

**业务目标**
- 将现场音频转换为多语种可消费内容（语音/字幕）。

**核心对象**
- `AudioInSession`：采集端会话。
- `TranscriptSegment`：ASR 片段（含时间戳、置信度）。
- `TranslationSegment`：译文片段（按语种）。
- `TTSChunk`：目标语种音频片段（可选）。

**关键规则**
- 片段必须具备可追踪 ID，用于重传与去重。
- 翻译链路按语种并行，任一语种失败不阻塞其他语种。
- 终端按语种订阅，仅接收目标语种增量数据。

**降级策略**
- TTS 异常：自动切换字幕优先。
- 弱网：降低码率/提高缓冲并给出提示。

**MVP 验收点**
- 单活动内支持至少 3 种目标语种并行输出。
- 断线后可自动重连并恢复订阅。

### 2.3 终端收听域（Audience Domain）

**业务目标**
- 听众通过手机/PC/iPad 快速加入并稳定收听。

**关键规则**
- 加入活动后必须先完成活动访问校验，再允许收听。
- 语种切换应在会话级生效，不要求页面刷新。
- 当语音不可用时，字幕视图自动置顶并持续滚动。

**降级策略**
- 重连期间展示最近可用字幕缓存，降低“完全无内容”时间。

**MVP 验收点**
- 主流浏览器可完成加入、切换语种、播放/字幕查看。

### 2.4 账号与权限域（Identity & Access Domain）

**业务目标**
- 支持 CWcomm 本地账号与气候周账号集成登录。

**角色模型（MVP）**
- `viewer`：仅可加入活动并收听。
- `operator`：可管理音频接入与运行态监控。
- `admin`：可管理活动、语种、成员权限。

**关键规则**
- 气候周账号首次登录：创建本地用户档案并建立外部 ID 映射。
- 同一外部账号仅绑定一个本地账号。
- 角色以“活动级授权”为准，全局角色仅做默认值。

**降级策略**
- 外部身份服务不可用时：
  - 已登录用户可在会话有效期内继续访问。
  - 新登录请求返回“稍后重试”并记录审计日志。

**MVP 验收点**
- 气候周账号可完成登录、登出、会话续期。
- 角色映射准确生效到活动资源权限。

### 2.5 外部集成域（Integration Domain）

**业务目标**
- 与气候周 OpenAPI 进行可控同步，避免脏数据覆盖。

**同步模式（MVP）**
- 定时增量同步（按时间窗口）。
- 管理员手动重同步（按活动 ID）。

**冲突处理规则**
- 外部来源字段优先级高于本地镜像字段。
- 本地可编辑字段需与外部字段隔离（例如本地运营备注）。
- 同步冲突进入待处理队列并提示管理员确认。

**MVP 验收点**
- 同步成功率、失败重试、冲突告警均可观测。

## 3. 关键业务规则清单

### 3.1 活动状态迁移规则

- `DRAFT -> READY`：完成最小配置（标题、时间、至少1个目标语种）。
- `READY -> LIVE`：音频输入已连通，健康检查通过。
- `LIVE -> ENDED`：管理员主动结束或到达活动结束时间。
- `ENDED -> ARCHIVED`：归档后只读。

### 3.2 活动来源与字段写入规则

- `source=local`：核心字段可由管理员维护。
- `source=climate_week_api`：外部托管字段只读，本地仅补充扩展字段。
- 同步写入必须记录 `lastSyncedAt`、`sourceVersion`、`syncStatus`。

### 3.3 登录绑定规则

- 外部唯一标识（如 `sub`）作为主绑定键。
- 发现绑定冲突时禁止自动合并，需人工处理。
- 登录与绑定失败必须产生日志事件，便于审计。

---

## 4. 功能验收标准（MVP）

- AC-01：管理员可在 2 分钟内完成活动创建并发布。
- AC-02：听众扫码后 10 秒内进入可收听状态（常规网络）。
- AC-03：活动 `LIVE` 期间可稳定输出多语种字幕。
- AC-04：任一语种翻译失败不影响其他语种输出。
- AC-05：网络短暂中断后可自动恢复并继续收听。
- AC-06：TTS 故障时字幕链路持续可用。
- AC-07：气候周活动同步失败触发重试并可被管理员感知。
- AC-08：气候周账号首次登录可自动建档并完成角色映射。
- AC-09：未授权用户无法进入受限活动。
- AC-10：关键操作（登录、同步、活动状态变更）可审计追踪。

## 5. 待架构设计落实的接口关注点

- 实时消息协议：片段 ID、时序对齐、重传机制。
- 身份令牌策略：会话时长、刷新机制、跨端一致性。
- 同步任务模型：幂等键、重试退避、死信队列。
- 可观测性埋点：延迟分段指标（采集/识别/翻译/分发）。
