# CWcomm 迭代实施计划（MVP）

> 状态：Draft v0.1（以文档冻结后进入执行）

---

## 1. 目标与节奏

- 目标：在 6~8 周内完成 MVP 开发、联调、试点。
- 迭代单位：按周推进，双周做一次里程碑评审。
- 约束：先打通“字幕优先闭环”，再增强语音链路。

---

## 2. 工作流与泳道

- `Backend/API`：活动、鉴权、同步、审计接口。
- `Realtime/AI`：采集、ASR、翻译、TTS、事件分发。
- `Frontend`：听众端、管理端、状态看板。
- `Data/Infra`：数据库迁移、缓存、队列、部署与监控。
- `QA/Release`：用例、回归、压测、试点发布。

---

## 3. 分阶段计划

### 阶段 0：冻结与启动（第 1 周）

交付目标：
- 冻结 `api-draft`、`data-model-draft`、`system-architecture` 的 MVP 范围。
- 确定外部依赖接入条件（OpenAPI、SSO、AI 服务）。

任务：
- Backend：接口字段与错误码评审。
- Data：DDL 初稿评审，索引与保留策略确认。
- Infra：环境拓扑与配置清单（dev/staging/prod）。
- QA：验收清单按 AC-01~AC-10 建立测试用例骨架。

里程碑出口：
- 文档冻结并形成变更流程（后续改动需评审）。

### 阶段 1：最小可运行骨架（第 2~3 周）

交付目标：
- 活动管理 + 鉴权 + 字幕事件最小链路打通。

任务：
- Backend：
  - 活动 CRUD/状态迁移接口。
  - SSO 登录回调、会话签发、角色校验中间件。
  - 审计日志与幂等中间件。
- Realtime：
  - `session.ready`、`subtitle.delta/final` 事件协议实现。
  - 断线重连 + `lastEventId` 补发。
- Frontend：
  - 听众端加入、语种切换、字幕显示。
  - 管理端活动创建/发布/结束页面。
- Data/Infra：
  - MVP 主表迁移 + Redis + 消息队列接入。

里程碑出口：
- 演示“单活动字幕闭环”（无 TTS 也可）。

### 阶段 2：集成与增强（第 4~5 周）

交付目标：
- 气候周 OpenAPI 同步与 SSO 端到端联调。
- 多语种并行翻译稳定运行。

任务：
- Backend：
  - 同步任务接口、任务状态查询、冲突处理接口。
  - 活动成员角色管理接口。
- Realtime/AI：
  - 多语种并发处理与隔离。
  - `pipeline.degraded` 降级事件。
- Frontend：
  - 管理端同步任务视图、冲突处理页。
  - 运行态看板（在线数、延迟、错误率）。
- QA：
  - 外部依赖故障注入（OpenAPI 限流、IdP 超时、TTS 故障）。

里程碑出口：
- 达成 AC-04、AC-06、AC-07、AC-08 的联调验收。

### 阶段 3：稳定性与试点（第 6~7 周）

交付目标：
- 完成性能压测、监控告警联通、灰度试点。

任务：
- Infra：
  - 指标埋点落地，仪表盘与告警规则上线。
  - 限流策略与弹性配置。
- Realtime：
  - 重连成功率优化、弱网降级策略调优。
- QA：
  - 500 并发压测与回归。
  - 安全用例（越权、token 过期、审计覆盖）。

里程碑出口：
- 达成 AC-02、AC-03、AC-05、AC-09、AC-10。

### 阶段 4：上线准备（第 8 周）

交付目标：
- 上线清单闭环与运维交接。

任务：
- 发布 Runbook、故障手册、回滚预案。
- 完成试点复盘与问题清零。
- 执行 go/no-go 评审。

里程碑出口：
- 具备正式上线条件。

---

## 4. 依赖与关键路径

关键路径：
1. `数据模型冻结` -> `活动/鉴权接口` -> `听众会话 + 字幕事件` -> `联调验收`
2. `外部接入凭据` -> `SSO/OpenAPI 集成` -> `冲突处理` -> `试点`

主要依赖：
- 气候周 OpenAPI 字段字典与 SLA。
- 气候周 IdP 的客户端凭据与回调配置。
- AI 服务配额与限流规则。

---

## 5. 风险与缓冲

- 外部系统联调延期：
  - 缓冲方案：先用 mock provider 完成本地闭环。
- TTS 成本或稳定性波动：
  - 缓冲方案：字幕优先策略默认开启。
- 并发压测不达标：
  - 缓冲方案：分层限流 + 语种级降载。

---

## 6. DoD（完成定义）

每个迭代故事必须满足：
- 代码 + 单测/接口测试通过。
- 指标与日志埋点齐全。
- 错误码、审计事件符合 `api-draft`。
- 至少 1 条回归用例进入自动化或手工清单。
- 文档更新（接口/模型变更必须同步文档）。

---

## 7. 建议任务拆分（首批）

1. EPIC-ACT：活动与权限（状态机、成员角色、加入码）。
2. EPIC-AUTH：SSO 与会话（登录回调、刷新、登出、绑定冲突）。
3. EPIC-RT：实时字幕链路（会话、事件、重连、ACK）。
4. EPIC-SYNC：OpenAPI 同步（任务、冲突、重试、告警）。
5. EPIC-OBS：监控与审计（SLO 指标、告警、审计查询）。
